LeetCode Problem (~10 minutes): Unique Paths

Problem Statement:

There is a robot on an m x n grid. The robot is initially located at the top-left corner (i.e., grid[0][0]). The robot tries to move to the bottom-right corner (i.e., grid[m - 1][n - 1]). The robot can only move either down or right at any point in time. Given the two integers m and n, return the number of possible unique paths that the robot can take to reach the bottom-right corner. The test cases are generated so that the answer will be less than or equal to 2 * 109.

Example 1:

Input: m = 3, n = 7
        Output: 28

Example 2:

Input: m = 3, n = 2
        Output: 3
        Explanation: From the top-left corner, there are a total of 3 ways to reach the bottom-right corner:
        1. Right -> Down -> Down
        2. Down -> Down -> Right
        3. Down -> Right -> Down

Solution

Since the robot can only move down and to the right, the number of unique paths to tile t is the sum of the number of unique paths to the tile to the left l of t and the number of unique paths to the tile above a of t. This is given by the following recurrence relation:

dp(i, j) = dp(i - 1, j) + dp(i, j - 1)
  1. Initialize a m x n 2D DP array dp, where dp[i][j] represents the unique number of paths to point (i, j).
  2. Set dp[0][0] = 1 since the number of unique paths to the starting position is 1.
  3. Iterate through every row i and column j in the grid and set dp[i][j] = dp[i - 1][j] + dp[i][j - 1].
  4. Return dp[m - 1][n - 1].

Code:

class Solution {
            public int uniquePaths(int m, int n) {
                int[][] dp = new int[m][n];
                dp[0][0] = 1;
                for (int i = 0; i < m; i++) {
                    for (int j = 0; j < n; j++) {
                        if (i == 0 || j == 0) {
                            dp[i][j] = 1;
                        } else {
                            dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
                        }
                    }
                }
                return dp[m - 1][n - 1];
            }
        }

Applying to Internships for Computer Science Students

When to Apply

For those eyeing summer internships, the application process usually kicks off the summer before. Here's a guide on when to apply:

  • Early Birds (July - September): Big tech companies start early.
  • Fall Recruiters (October - December): Many companies open applications in the fall.
  • Winter Window (January - March): Late planners and second-round openings.
  • Spring Stragglers (April - June): Few companies start their search in the spring.

Best Application Practices

  1. Tailor Your Resume: Customize your resume for each application.
  2. Track Your Applications: Use tools like Huntr to keep track of applications.
  3. Cover Letter: Tailor it to the company and position.
  4. Practice Coding Interviews: Use platforms like LeetCode and HackerRank.
  5. Research Companies: Understand each company's culture and projects.
  6. Networking: Connect with current and former interns on LinkedIn.

Great Resources for Applying

  • GitHub Repositories: Repositories dedicated to internship applications.
  • Handshake, Glassdoor, LinkedIn: Central hubs for job openings and networking.
  • Internship Aggregators: Websites like Internships.com and WayUp.

Applying With a Referral

Getting referred by a current employee can significantly increase your chances of landing an interview. Here’s how to increase your chances of getting a referral:

  1. Network at industry meetups and career fairs.
  2. Utilize your university’s alumni networks.
  3. Reach out to professionals on LinkedIn with a thoughtful message.

Career Fairs

Career fairs are opportunities to meet recruiters face-to-face. Prepare your pitch, research participating companies, bring copies of your resume, and follow up with a thank you email.

Starting early, staying organized, and leveraging resources can make the internship application process less daunting and more successful. Persistence and preparation are key.
